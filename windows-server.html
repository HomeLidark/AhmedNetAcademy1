👥 Active Directory & Domain Controller

إدارة المستخدمين والأجهزة في المؤسسة

ما هو Domain Controller؟

DC هو الخادم الذي يحتوي على قاعدة بيانات Active Directory. يتحكم في:

  • تسجيل دخول المستخدمين
  • سياسات الأمان (GPO)
  • مصادقة الأجهزة
  • خدمات DNS و DHCP
نصيحة: في المؤسسات، يُنصح بوجود DC ثانوي (Backup DC) لتجنب التوقف.
خطوات رفع DC
  1. تثبيت Windows Server
  2. تثبيت دور Active Directory Domain Services
  3. تشغيل معالج Active Directory Domain Services Configuration
  4. اختيار Add a new forest (مثال: company.local)
  5. تعيين كلمة مرور DSRM
  6. إعادة التشغيل
# تثبيت AD DS عبر PowerShell
Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools
Install-ADDSForest -DomainName "company.local"
الخدمات الأساسية

🌐 DNS & DHCP

حل الأسماء وتوزيع العناوين تلقائياً

DNS (Domain Name System)

يحول أسماء النطاقات (مثل www.company.local) إلى عناوين IP.

  • Forward Lookup Zone: اسم → IP
  • Reverse Lookup Zone: IP → اسم
  • Record Types: A, CNAME, MX, SRV
# إنشاء سجل A
Add-DnsServerResourceRecordA -ZoneName "company.local" -Name "mail" -IPv4Address "192.168.1.10"
DHCP (Dynamic Host Configuration Protocol)

يوزع عناوين IP تلقائياً على الأجهزة عند اتصالها بالشبكة.

  • Scope: نطاق العناوين المتاحة
  • Reservation: تثبيت عنوان IP لجهاز معين
  • Lease Time: مدة استئجار العنوان
# تثبيت DHCP
Install-WindowsFeature -Name DHCP -IncludeManagementTools
Add-DhcpServerv4Scope -Name "LAN" -StartRange "192.168.1.100" -EndRange "192.168.1.200" -SubnetMask "255.255.255.0"
التحكم الكامل

⚙️ Group Policy (GPO)

تطبيق سياسات على المستخدمين والأجهزة دفعة واحدة

سياسات الأمان
  • تعقيد كلمة المرور
  • قفل الحساب بعد 3 محاولات
  • تغيير كلمة المرور كل 30 يوماً
تخصيص سطح المكتب
  • تثبيت خلفية موحدة
  • إخفاء أيقونات معينة
  • تعيين شاشة توقف
القيود
  • منع تشغيل الـ USB
  • منع تثبيت البرامج
  • منع الوصول إلى لوحة التحكم
أوامر GPO الشائعة
# تحديث GPO يدوياً
gpupdate /force

# معرفة GPO المطبقة
gpresult /r
# Backup GPO
Backup-GPO -Name "Security Policy" -Path "C:\GPOBackup"

# Restore GPO
Restore-GPO -BackupId "{GUID}" -TargetName "Restored Policy"
المشاركة والطباعة

📁 File & Print Server

مشاركة الملفات والطابعات في المؤسسة

File Server
  • NTFS Permissions: صلاحيات على مستوى الملفات والمجلدات
  • Share Permissions: صلاحيات على مستوى المشاركة
  • DFS (Distributed File System): توحيد مجلدات متعددة
# إنشاء مشاركة مجلد
New-SmbShare -Name "SharedData" -Path "C:\Data" -FullAccess "DOMAIN\Admins"
Print Server
  • إدارة الطابعات من مكان واحد
  • توزيع برامج التشغيل تلقائياً
  • مراقبة استخدام الطابعات
# تثبيت Print Server
Install-WindowsFeature -Name Print-Services -IncludeManagementTools

# مشاركة طابعة
Add-Printer -Name "HP-LaserJet" -ShareName "HP-Print"
المحاكاة الافتراضية

🖥️ Hyper-V

إنشاء وإدارة الأجهزة الافتراضية على Windows Server

Virtual Machines

تشغيل أنظمة تشغيل متعددة على خادم واحد.

Checkpoints

إنشاء نقاط استعادة قبل إجراء تغييرات كبيرة.

Live Migration

نقل الأجهزة الافتراضية بين الخوادم دون توقف.

أوامر Hyper-V في PowerShell
# إنشاء VM
New-VM -Name "Windows-Client" -MemoryStartupBytes 4GB -BootDevice VHD -VHDPath "C:\VMs\client.vhdx" -Path "C:\VMs" -Generation 2

# تشغيل VM
Start-VM -Name "Windows-Client"

# إيقاف VM
Stop-VM -Name "Windows-Client"
